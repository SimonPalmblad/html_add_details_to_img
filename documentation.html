1 Getting Started

1.1 What is Sonity?

Sonity 是一个音频中间件，让你在 Unity 中完全掌控你的声音。它在易用性与高级功能之间取得平衡，帮助你快速让游戏听起来很出色。

  • 1.1.0 新功能： Steam Audio 与 PlayMaker 集成。
  • 1.0.6 新功能： 可将声音音量提升至 0dB 以上。
  • 提供无需编写代码即可触发声音的组件。
  • 使用可编写脚本对象（ScriptableObject）实现实时与批量编辑。
  • 通过 Sound Parameters 在运行时控制声音。
  • 自动创建资源并查找引用。
  • 支持音乐功能，例如播放列表（Playlists）、分轨（Stems）和前奏（Intros）。
  • 可在游戏运行时可视化调试声音。
  • 每个声音通道（Voice）均支持自定义 DSP 滤波器和失真效果。
  • 高度优化的系统，支持 IL2CPP。
  • 提供完整、文档齐全的源代码。
  • 已在商业游戏中开发与使用。
  • 由拥有游戏音频专业经验的作者精心打造。

许可说明：

Sonity 在 Tuanjie 资源商店（Tuanjie  Asset Store）上的许可为单用户授权。

如果有多位用户需要使用 Sonity，则必须为每位用户单独购买许可。

1.3 安装

购买后，前往 Package Manager（包管理器），显示 “My Assets（我的资源）”，然后搜索 “Sonity”。

接着点击 “Sonity”，按下 “Download（下载）”，然后再点击 “Import（导入）”。

安装完成后，Sonity 应位于路径 Assets\Plugins\Sonity。

如果材质显示为粉色，请选中所有 “ExampleMaterial”，并通过以下路径升级到 URP：

Edit → Rendering → Materials → Convert Selected Built-in Materials to URP

1.4 Adding the SoundManager

The SoundManager is the master object which is used to play sounds and manage global settings for your project.

An instance in the scene is required to play SoundEvents.

You can add the pre-made prefab called “SoundManager” found in “Assets\Plugins\Sonity\Prefabs” to the scene.

Or you can add the “Sonity - Sound Manager” component to an empty GameObject in the scene, it works just as well.

❕ Tip: Don’t forget to set up the Distance Scale to match the scale of your project.

The SoundManager has options for controlling the volume, playback, and fine tuning spatialization via distance scaling. These can be controlled directly on the GameObject.

Sound Manager Settings

Enable Volume Increase

When enabled, you get the ability to raise the volume of SoundContainers and SoundEvents by up to +24dB each and SoundVolumeGroups by up to +12dB.

You also need to set the Scripting Define Symbol "SONITY_ENABLE_VOLUME_INCREASE".

This can be done by pressing the button that appears after clicking the Enable Volume Increase checkbox (✅)

Global Volume

Sets the global volume of all Sonity and Unity AudioSources using AudioListener.volume

Global Pause

Pauses or unpauses all sound using AudioListener.pause.

Except any SoundEvents set to "Ignore Global Pause".

Global SoundTag

The selected Global SoundTag.

SoundEvents using global SoundTag are affected by this.

Distance Scale

Global range scale multiplier for all the sounds in Sonity.

Distance is calculated using Tuanjie units of distance.

E.g. if Distance Scale is set to 100, a SoundEvent with the distance multiplier of 1 will be heard up to 100 Unity units away.

Third Party Integrations

Sonity has integrations with Steam Audio and PlayMaker. These require you to have access to the packages yourself. You can then enable the integration in your Sound Manager.

More info at SoundManager Component.

1.5 Audio playback and customization

Alongside a SoundManager, you also need three more components to play sound in Sonity:

(AudioClip > SoundContainer > SoundEvent > Play sound)

AudioClips are built-in file types for handling and playing sound in the Tuanjie Engine.

SoundContainers and SoundEvents are Sonity’s custom assets that expand the functionality of the AudioClip.

They control how the audio is played,enabling control over features such as spatialization, filters, and volume, and more which will be detailed later.

(Lägga till liknelse?)[b]

Since they are made of scriptable objects, all changes you make in runtime are updated live and saved, making it easy to demo and make changes live inside your game.

All Sonity scriptable objects and components are also multi-object editable.

1.5.1 Asset Creation

Since Sonity relies on several components for audio playback, there are tools to simplify and speed up workflow built into the plugin.

  • To create both SoundContainers and SoundEvent assets from AudioClips, you can just select them (or their folder).
  • Right-click to bring up the Create → Sonity → Create Assets from Selection-menu and choose the option you need.

Below is an example of creating a SoundContainer for each AudioClips with a single SoundEvent using the option SE + SC from AudioClip Single SE.

Beside quickly creating assets the context menu actions also provide automatic naming convention detection and loop functionality:

  • If a SoundContainers name contains “loop” it will  automatically enable “Loop”, “Follow Position”, “Stop if Transform is Null” and “Random Start Position”.
  • It will automatically find out if you use space, hyphen “-” or underscore “_” in your filenames and will style the file ending the same way (e.g. “_SC”, “-SC” etc.)

SoundEvents

SoundEvents are used to control audio playback in Sonity.

They can contain one or more SoundContainers.

Multi-object editable.

SoundEvents feature:

  • A timeline for editing and creating sound composites.
  • Handle fade-in and fade-out of the audio.
  • Controlling parameters via intensity, which enables you to control one or several options within the SoundEvent via this one parameter.
  • Trigger another SoundEvent on Play, Stop, Tail or Tag.
  • Create composite audio loops with customized start and end sounds. Useful for audio with a different start, sustain and ending.

1.8 Playing and Stopping Sounds with SoundTrigger Basics

The easiest way to play a SoundEvent is to create a gameobject and add a SoundTrigger component to it.

Built-in trigger events: A SoundTrigger is a component used for playback of SoundEvent(s). They rely on the built-in callbacks in Tuanjie - Enable, Disable, OnCollisionEnter etc.

Easy distance control: SoundTriggers allow for easy adjustments of how far the SoundEvent should be heard, which is editable in the scene viewport through the radius handle.

1.9 Playing and Stopping Sounds with C# Basics

If you want to control your audio directly via code there are many options.

  • To play a SoundEvent, create a C# script and include the “Sonity” namespace.
  • Make a public “SoundEvent” property, select the desired SoundEvent in the inspector.
  • Use the functions of the SoundEvent to play and stop your SoundEvents

Example code:

using UnityEngine;
using Sonity;

public class PlayStopExample : MonoBehaviour {

   public SoundEvent soundEvent;

   void PlayExample() {
       // Plays the SoundEvent at the position of the transform
       soundEvent.Play(transform);
   }

    void StopExample() {
       // Stops the SoundEvent playing at the transform
       soundEvent.Stop(transform);
   }
}

1.10 Templates & Examples

Sonity contains examples and script templates of advanced functionality which you can use in your game.

Example_AudioMixer.mixer

An example AudioMixer with groups for Ambience, VO, SFX, Music, and UI.

TemplateSoundMusicManager.cs

Template of a singleton music playback system.

Add the script to a GameObject in the scene and use like this:

SonityTemplate.TemplateSoundMusicManager.Instance.PlayMainMenu();

TemplateSoundPlayUI.cs

Template of a singleton used to play e.g. UI sounds.

Useful when you want to play SoundEvents in a lot of places through code.

Add the script to a GameObject in the scene and use like this:

SonityTemplate.TemplateSoundPlayUI.Instance.PlayButtonClick();

TemplateSoundVolumeManager.cs

Template of a singleton AudioMixer volume controller.

The default settings work with the provided "Example_AudioMixer.mixer".

Add the script to a GameObject in the scene and use like this:

SonityTemplate.TemplateSoundVolumeManager.Instance.SetVolumeMaster(1f);
